# Contribución Final

Hoy se convierten en contribuidores open source.

---

## Repaso rápido

El flujo completo:

```
1. Sincronizar fork
2. Crear rama
3. Hacer cambios
4. Commit y push
5. Abrir PR
6. Code review
7. Aprobación
8. Merge 🎉
```

Note:
Repaso rápido de lo aprendido en 5 clases.
Hoy lo ponen todo en práctica.

---

## Elegir un issue

Opción 1: Tomar un issue existente.
Opción 2: Implementar una mejora que encontraron.
Opción 3: Algo que siempre quisieron agregar.

Cómanse algo pequeñito pero real.
No tiene que ser enorme para ser valioso.

Note:
Ayudar a cada estudiante a encontrar un issue adecuado.
Si alguien no encuentra, asignarle algo pequeño.

---

## Tiempo de contribución

45 minutos para:

- Elegir issue.
- Sincronizar fork.
- Crear rama.
- Implementar.
- Commit y push.
- Abrir PR.
- Responder a code review.

Yo voy a ir pasando por los asientos.

Note:
Dar tiempo. Ayudar individualmente.
Los que terminen, que ayuden a otros.

---

## Fiesta de Merge

Cada PR que se mergea...

```
🎉 Felicitaciones, [nombre]!
   Tu primer PR open source está mergeado.
   Eres oficialmente contribuidor.
```

Vamos a celebrar cada merge con un aplauso.

Note:
Crear ambiente festivo.
Capturar pantalla de cada merge.

---

## Eres contribuidor open source

Tu nombre aparece en el historial del proyecto.

```
git log --all --oneline
```

Tu perfil de GitHub muestra la contribución.

```
github.com/TU_USUARIO
→ Contribution activity
```

Esto es para siempre. Así consigues trabajo.

Note:
Mostrar cómo se ve la contribución en el perfil.
Es literalmente prueba de experiencia laboral.

---

## Cómo seguir contribuyendo

Esto no termina aquí.

- Sigue el proyecto en GitHub (Watch).
- Vuelve cuando veas algo que mejorar.
- Contribuye a otros proyectos.
- Abre issues cuando encuentres bugs.

El open source es un hábito, no un evento.

Note:
Dar recursos para seguir: opensource.guide, GitHub Explore.
Motivar a seguir contribuyendo.
