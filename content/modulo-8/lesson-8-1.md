# Open Source

El software que mueve al mundo.

---

## ¿Qué es open source?

Código fuente abierto y disponible para todos.

- Cualquier persona puede verlo.
- Cualquier persona puede usarlo.
- Cualquier persona puede mejorarlo.
- Cualquier persona puede compartirlo.

Note:
Preguntar: ¿Qué ejemplos de software open source conocen?
Linux, React, VS Code, Python, WordPress.

---

## Open Source vs Código Cerrado

| Open Source | Código Cerrado |
|---|---|
| Público | Privado |
| Comunidad | Empresa |
| Gratuito | Pagado |
| Colaborativo | Jerárquico |
| Transparente | Caja negra |

Note:
Ejemplos de código cerrado: Windows, Photoshop, Microsoft Office, iOS.

---

## ¿Por qué contribuir?

✓ Tu código en producción.
✓ Portafolio real para entrevistas.
✓ Aprendes de developers experimentados.
✓ Conoces código de calidad profesional.
✓ Construyes tu reputación en GitHub.
✓ Das algo de vuelta a la comunidad.

Preguntar: ¿Cuántos de ustedes tienen proyectos en GitHub?

## El flujo de contribución

```
1. Encontrar un proyecto
2. Hacer Fork
3. Clonar localmente
4. Crear rama
5. Hacer cambios
6. Commit y Push
7. Abrir Pull Request
8. Code Review
9. Aprobación
10. Merge
```

Note:
Mostrar el diagrama completo.
Hoy nos enfocamos en entender este flujo.
Las próximas clases lo harán paso a paso.

## Código de conducta

Todo proyecto open source tiene reglas.

- Sea respetuoso.
- Critique el código, no a la persona.
- Ayude a otros contribuidores.
- Siga las guías de contribución.

Note:
Explicar que el código de conducta no es opcional.
Es obligatorio para participar en la comunidad.
